# Device Catalog

The `devices.yaml` file at the repository root is an auto-generated catalog that provides a manufacturer-centric view of all supported devices and their drivers.

## Purpose

While `index.yaml` is a flat driver registry optimized for gateway consumption, `devices.yaml` answers the question: *"I have a device from manufacturer X, model Y — which driver do I need?"*

## Generation

Generated by `tools/generate_devices.py` from all manifest files:

```bash
uv run tools/generate_devices.py
```

Do not edit `devices.yaml` manually. CI validates that it stays in sync with manifests.

## Schema

```yaml
version: 1
manufacturers:
  - name: "Huawei"                     # Manufacturer name
    model_families:
      - name: "SUN2000"               # Model family / product line
        variants: [SUN2000-10KTL-M1]  # Specific models tested
        regions: [EU]                  # Regional variants
        protocols:                     # Available driver(s)
          - protocol: modbus
            driver: "huawei"           # Driver name (maps to drivers/{name}.lua)
            version: "1.1.0"
            ders: [battery, meter, pv]
            control: false
        firmware_versions: ""
        notes: ""
```

## Key Concepts

### Manufacturer → Model Family → Protocol

The hierarchy groups devices by manufacturer, then by model family. When the same device supports multiple protocols (e.g., Modbus TCP and MQTT), each protocol variant is a separate driver listed under `protocols`.

### Multiple Protocols

Some manufacturers offer devices accessible via different protocols. These are implemented as separate drivers but appear together in the catalog:

```yaml
  - name: "Ferroamp"
    model_families:
      - name: "EnergyHub"
        protocols:
          - protocol: mqtt
            driver: "ferroamp"
          - protocol: modbus
            driver: "ferroamp_modbus"
```

### Regions

The `regions` field indicates where a device variant has been tested. Common values: `EU`, `CN`, `INT` (international), `US`, `AU`. Empty list means no region-specific testing data.

### Variants

The `variants` field lists specific model numbers within a family. For example, the Sungrow SH-RT family includes SH5.0RT, SH6.0RT, SH8.0RT, and SH10RT.
